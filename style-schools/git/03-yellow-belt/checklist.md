# Git Yellow Belt Checklist

This checklist outlines the concepts and tasks required to achieve your Git Yellow Belt.

## Core Concepts

- [ ] Understanding remote repositories (origin, upstream)
- [ ] Push and pull operations
- [ ] Remote branches and tracking branches
- [ ] Pull requests and code review workflow
- [ ] Merge conflicts (recognition and resolution)
- [ ] Rebasing vs merging strategies
- [ ] Cherry-picking commits
- [ ] Git stash for temporary changes
- [ ] Working with forks
- [ ] GitHub/GitLab workflow (issues, PRs, reviews)

## Practical Tasks

- [ ] Clone a repository and set up multiple remotes
- [ ] Create a feature branch, push it, and open a pull request
- [ ] Resolve a merge conflict between two branches
- [ ] Rebase a feature branch onto main
- [ ] Use `git stash` to save and restore work in progress
- [ ] Cherry-pick a specific commit from one branch to another
- [ ] Review a pull request and leave comments
- [ ] Fork a repository and contribute via pull request
- [ ] Sync a fork with upstream changes
- [ ] Use `git log` with various options to view history

## Projects

- [ ] Complete a full pull request workflow on a real project
- [ ] Contribute to an open source repository
- [ ] Maintain a clean commit history across multiple feature branches

Once you can confidently check off these items, you are ready for the Orange Belt!
