# Fintech Green Belt Checklist

## Core Concepts

- [ ] Payment orchestration (routers, retries, fallbacks)
- [ ] Double-entry ledgers and reconciliation
- [ ] Idempotency, sequencing, and exactly-once semantics
- [ ] Settlement cycles, FX handling, and liquidity
- [ ] Fraud detection (rules + ML) and risk scoring
- [ ] Compliance (KYC/AML, PCI scope, privacy)
- [ ] Webhooks, signatures, and replay protection
- [ ] Observability for money movement (SLOs, audit trails)
- [ ] High-availability patterns and disaster recovery
- [ ] Performance under peak load (Black Friday scenarios)

## Practical Tasks

- [ ] Implement idempotent payment APIs with idempotency keys
- [ ] Design double-entry ledger with daily reconciliation
- [ ] Build payment router with retries and fallback providers
- [ ] Add webhook signing, timestamping, and replay defense
- [ ] Create fraud rules pipeline with feature logging
- [ ] Model settlement windows and fund availability
- [ ] Implement dispute and chargeback workflows
- [ ] Add SLOs for auth, capture, refund latency
- [ ] Build dashboards for success rate and decline codes
- [ ] Document compliance boundaries and data handling

## Projects

- [ ] Payment orchestration system with multi-PSP routing
- [ ] Ledger-backed wallet with reconciliation and audits
- [ ] Fraud detection + dispute management service

Once you can confidently check off these items, you are ready for the Blue Belt!
